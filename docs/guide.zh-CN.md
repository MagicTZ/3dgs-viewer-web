# 使用指南

[English](./guide.md) | 简体中文

本文档集中说明本项目的启动方式、交互操作、镜头规划逻辑、删除编辑流程与视频导出说明。顶层 `README.md` 负责项目概览与快速上手，这里保留完整使用手册。

## 1. 项目概览

`3DGS Viewer` 是一个基于 `Spark.js` 和 `Three.js` 的浏览器端 3D Gaussian Splatting 查看与轻编辑工具，当前版本主要聚焦于两条主线：

- 固定中心点的镜头规划与 MP4 导出
- 浏览器内的 splat 删除编辑与 `.ply` 保存

这版已经不是“手动拖相机再记关键帧”的旧流程，而是镜头点规划版本：

- 双击模型设置固定镜头中心 `Pivot`
- 用离散镜头点规划路径
- 所有镜头点和中间采样帧始终朝向 `Pivot`
- `P` 预览和导出视频走同一套路径采样逻辑

## 2. 环境要求与启动

### 环境要求

- Chrome / Edge，且浏览器支持 `WebCodecs`
- 通过 HTTP 服务器访问，不能直接双击 `index.html`

## 3. 推荐使用流程

### 3.1 上传模型

- 页面初始进入是空态，不会自动加载仓库中的固定模型
- 点击左侧 `Open File` 或中心空态按钮，选择本地 3DGS 文件
- 也可以直接把模型文件拖到页面中
- 支持格式：`.ply` / `.splat` / `.spz` / `.ksplat`

### 3.2 浏览场景

- 左键拖拽旋转视角
- 右键拖拽平移
- 滚轮缩放
- 方向键做视角转向

### 3.3 设置镜头中心

- 双击模型，设置固定镜头中心 `Pivot`
- 设置后，镜头规划、路径预览、视频导出都围绕这个中心工作
- 普通平移和缩放不会再改动 `Pivot`

### 3.4 进入镜头规划

- 点击左上角 `进入规划`
- 初始不会自动显示默认镜头点
- 进入规划后，按 `+` 会从当前相机机位插入一个镜头点

### 3.5 组织镜头路径

- 点击镜头点可切到该点视角预览
- `+`：在当前选中点后插入新点；未选中时追加到末尾
- `Del`：删除当前镜头点
- `C`：清空所有镜头点
- `P`：按镜头点顺序预览整条路径

注意：

- 至少需要 `2` 个镜头点，才能播放和导出
- 所有镜头点默认始终朝向中心点
- 两个镜头点之间的中间采样帧也会持续 `lookAt(Pivot)`

### 3.6 导出视频

- 右上角设置分辨率、FPS、总时长、码率
- 点击 `导出 MP4 视频`
- 导出使用离屏渲染路径，和预览共享同一套路径采样逻辑

## 4. 交互说明

### 4.1 导航

| --- | --- |
| 点击 `Open File` | 选择本地 3DGS 模型 |
| 拖拽文件到页面 | 加载本地 3DGS 模型 |

| 右键拖拽 | 平移 |
| 滚轮 | 缩放 |
| 方向键 | 旋转 / 俯仰视角 |
| `R` + 左键拖拽 | 围绕当前中心做环绕观察 |
| 双击模型 | 设置或重设固定镜头中心 |

### 4.2 镜头规划

| 操作 | 说明 |
| --- | --- |
| `进入规划` | 切到镜头规划模式 |
| 点击镜头点 | 选中并预览该镜头点 |
| `+` | 从当前机位插入镜头点 |
| `Del` | 删除当前镜头点 |
| `C` | 清空所有镜头点 |
| `P` | 播放 / 停止路径预览 |

### 4.3 删除编辑

| 操作 | 说明 |
| --- | --- |
| `E` | 进入 / 退出删除编辑模式 |
| `1` | Picker |
| `2` | Brush |
| Picker 左键点击 | 单点选择 |
| Picker 左键拖拽 | 框选 |
| Brush 左键拖拽 | 刷选 |
| `Shift` | 加选 |
| `Ctrl/Cmd` | 减选 |
| `[` / `]` | 调整画笔半径 |
| `Esc` | 清空选择 |
| `Del` | 删除选中的 splats |
| `Ctrl+Z` | 撤销删除 |
| `Ctrl+Y` | 重做删除 |
| `Ctrl+S` | 导出当前可见 splats 为 `.ply` |

删除键语义和模式有关：

- 镜头规划模式下，`Del` 删除镜头点
- 删除编辑模式下，`Del` 删除 splats

## 5. 核心机制

### 5.1 固定中心点镜头规划

当前镜头系统基于：

- `shotPivot`：固定中心点
- `shotPoints`：镜头点数组

每个镜头点存的是参数化位置，而不是手调四元数：

- `radius`
- `azimuth`
- `height`

真实相机位置由 `shotPivot + 极坐标参数` 实时计算，相机朝向统一通过 `lookAt(shotPivot)` 生成。

### 5.2 预览与导出一致

`P` 预览和 MP4 导出都消费同一套由镜头点派生出的路径，因此：

- 路径顺序一致
- 采样方向一致
- 预览和导出画幅逻辑一致

导出时会自动隐藏编辑辅助元素，避免把操作标记带进视频。

### 5.3 删除编辑与保存

删除编辑不修改原始源文件，而是在运行时隐藏对应 splats。保存时会把当前可见内容重新写出为一个新的 `.ply` 文件。

适合用来：

- 去掉漂浮噪点
- 裁掉场景边缘杂点
- 保存一个更干净的展示版本

## 6. 模型与格式

当前版本不再依赖固定的 `./model.ply` 初始模型，而是由用户在页面中主动选择或拖拽本地文件。

当前上传入口支持以下常见 3DGS 格式：

- `.ply`
- `.splat`
- `.spz`
- `.ksplat`

模型切换成功后，会自动重置当前镜头规划和删除编辑状态，并重新对焦到新模型。

## 7. 已知限制

- 当前是纯静态前端页面，没有后端
- 当前只支持本地上传，不支持服务端存储或 URL 载入
- 目前没有镜头点数值编辑器，镜头点主要通过当前机位插入
- 截图与示例视频资源仍可继续补充

## 8. 二次开发建议

- 加一个构建流程，例如 `Vite`
- 将 `viewer.js` 拆成导航、镜头规划、编辑导出等模块
