# 3DGS Viewer

一个基于 `Spark.js` 和 `Three.js` 的 3D Gaussian Splatting 网页查看器，当前版本聚焦于两条主线：

- 固定中心点的镜头规划与 MP4 导出
- 浏览器内的 splat 删除编辑与 `.ply` 保存

这版已经不是“手动拖相机再记关键帧”的旧流程，而是 `v1.0` 的镜头点规划版本：

- 双击模型设置固定镜头中心 `Pivot`
- 用离散镜头点规划路径
- 所有镜头点和中间采样帧始终朝向 `Pivot`
- `P` 预览和导出视频走同一套路径采样逻辑

## 当前能力

- 加载 3DGS 场景并自动对焦
- 运行时将场景主平面对齐到固定世界坐标系，减少旋转语义混乱
- 双击模型设置固定镜头中心
- 镜头规划面板：插点、删点、清空、预览
- 镜头点编号、朝向箭头、连线、路径线可视化
- 预览与导出共用同一条镜头路径
- Picker / Brush 两套 splat 删除编辑方式
- 撤销 / 重做 / 保存当前可见 splats 为 `.ply`

## 快速开始

### 环境要求

- Chrome / Edge，且浏览器支持 `WebCodecs`
- 通过 HTTP 服务器访问，不能直接双击 `index.html`

### 文件准备

当前代码默认读取根目录下的 `model.ply`，并且 `.gitignore` 已经忽略了这个文件。

目录示例：

```text
3dgs-viewer-web/
├── index.html
├── viewer.js
├── README.md
├── LICENSE
└── model.ply
```

### 启动方式

```bash
python -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

## 推荐使用流程

### 1. 浏览场景

- 左键拖拽旋转视角
- 右键拖拽平移
- 滚轮缩放
- 方向键做视角转向

### 2. 设置镜头中心

- 双击模型，设置固定镜头中心 `Pivot`
- 设置后，镜头规划、路径预览、视频导出都围绕这个中心工作
- 普通平移和缩放不会再改动 `Pivot`

### 3. 进入镜头规划

- 点击左上角 `进入规划`
- 初始不会自动显示默认镜头点
- 进入规划后，按 `+` 会从当前相机机位插入一个镜头点

### 4. 组织镜头路径

- 点击镜头点可切到该点视角预览
- `+`：在当前选中点后插入新点；未选中时追加到末尾
- `Del`：删除当前镜头点
- `C`：清空所有镜头点
- `P`：按镜头点顺序预览整条路径

注意：

- 至少需要 `2` 个镜头点，才能播放和导出
- 所有镜头点默认始终朝向中心点
- 两个镜头点之间的中间采样帧也会持续 `lookAt(Pivot)`

### 5. 导出视频

- 右上角设置分辨率、FPS、总时长、码率
- 点击 `导出 MP4 视频`
- 导出使用离屏渲染路径，和预览共享同一套路径采样逻辑

| `C` | 清空所有镜头点 |

### 删除编辑

| 操作 | 说明 |
| --- | --- |
| `E` | 进入 / 退出删除编辑模式 |
| `1` | Picker |
| `2` | Brush |
| Picker 左键点击 | 单点选择 |
| Picker 左键拖拽 | 框选 |
| Brush 左键拖拽 | 刷选 |
| `Shift` | 加选 |

- 删除编辑模式下，`Del` 删除 splats

## 功能结构

### 1. 固定中心点镜头规划

当前镜头系统基于：

- `shotPivot`：固定中心点
- `shotPoints`：镜头点数组

每个镜头点存的是参数化位置，而不是手调四元数：

- `radius`
- `azimuth`
- `height`

真实相机位置由 `shotPivot + 极坐标参数` 实时计算，相机朝向统一通过 `lookAt(shotPivot)` 生成。

### 2. 预览与导出一致

`P` 预览和 MP4 导出都消费同一套由镜头点派生出的路径，因此：

- 路径顺序一致
- 采样方向一致
- 预览和导出画幅逻辑一致

导出时会自动隐藏编辑辅助元素，避免把操作标记带进视频。

### 3. 删除编辑与保存

删除编辑不修改原始源文件，而是在运行时隐藏对应 splats。保存时会把当前可见内容重新写出为一个新的 `.ply` 文件。

适合用来：

- 去掉漂浮噪点
- 裁掉场景边缘杂点
- 保存一个更干净的展示版本

## 模型路径与格式

当前默认模型路径写在 [viewer.js](./viewer.js) 中：

```js
const splats = new SplatMesh({
  url: "./model.ply",
});
```

如果你要换成别的文件名或路径，直接改这里即可。

`Spark.js` 本身支持多种 3DGS 相关格式，但这个示例仓库当前默认按 `model.ply` 组织。
